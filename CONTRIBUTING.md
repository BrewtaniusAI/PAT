# Contributing to PAT

PAT is public, patent-free infrastructure. Contributions should improve reproducibility, clarity, language support, or safety.

## Contribution priorities
- language profiles
- tokenizer improvements
- documentation
- dataset annotation guides
- validation and tests

## Rules
- keep changes deterministic where possible
- include tests for behavior changes
- avoid adding opaque dependencies
- preserve cultural context in language additions

## Pull request checklist
- [ ] code added or updated
- [ ] tests added or updated
- [ ] docs updated
- [ ] no policy bypass introduced
